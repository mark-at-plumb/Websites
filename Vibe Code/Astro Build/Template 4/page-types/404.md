# 404 Page — Template Spec

This doc covers the 404 (not-found) page on every site in the network. Like the privacy policy and terms of use, this is a **template-control spec, not a Writer subagent brief**. The same generic 404 renders across all 100 sites with only per-site values injected.

Cross-references: `brand-voice-guide.md` (voice), `seo-foundations.md` (Part B10 — 404 handling), `content-quality-rules.md`.

## 1. Job-to-be-done

Recover conversions from broken URLs. The 404 has one job: get the user from "this page does not exist" to "I have called the plumber" as fast as possible. Phone is the answer.

The page must return a proper HTTP 404 status (not a soft 404 — see `seo-foundations.md` Part B10).

## 2. Search intent

None. 404 pages are not indexed and do not receive search traffic. The user arrived from a broken link, a mistyped URL, or stale bookmark.

## 3. Target queries

None.

## 4. Word count

**~100-150 words total.** Short on purpose. Every word that delays the user from calling is a lost conversion.

## 5. Per-site variable fields (template injects these)

The same generic 404 renders on every site. Only these fields vary:

| Field | Source |
|---|---|
| Site name | `site.siteName` |
| Phone | `site.trackingPhone` |
| Home URL | `/` (always the same) |

No suburb, no adjoining suburbs, no service-specific framing. Generic.

## 6. Page structure (in order)

### 6.1 H1

**Pattern:** `Page Not Found`

No suburb in the H1. Generic across all 100 sites.

### 6.2 Phone block (primary CTA — above the fold)

**Layout:** rendered by the template as a prominent phone CTA card. Sits directly below the H1. The largest visual element on the page.

**Content:**
- Phone number from `site.trackingPhone`, displayed in click-to-call format on mobile
- Headline: `Call Now | 24/7 Emergency Plumber`
- Single supporting line (~20 words): "Need a plumber? Skip the search and call us directly. We answer 24/7 across Adelaide."

### 6.3 Body (1 short paragraph, ~50-70 words)

Matter-of-fact, no over-apologising. Plain explanation that the page does not exist and what to do next.

```
The page you are looking for does not exist or has been moved. If you are looking for a plumber, call [site.siteName] on [site.trackingPhone] for a 24/7 response. Or head back to our home page.
```

### 6.4 Single "Back to Home" link

One link, prominent. No menu of options.

```
[Back to Home →](/)
```

## 7. Schema requirements

- **Plumber LocalBusiness** with full schema (template lifts from site-config — same as every page)
- **No BreadcrumbList** (404s do not have a position in the site hierarchy)

The page returns HTTP 404. The template must not return HTTP 200 for a not-found page (soft 404).

## 8. Internal linking

- **One link back to Home** in the body
- **Sticky mobile header phone CTA** (template default)
- **Site-wide footer** (template default — includes the standard footer nav, transparency line, ABN, licence)

No other body links. The 404 is intentionally minimal to keep the user moving toward the phone or the home page.

**Critical:** zero cross-linking between sites in the 100-microsite network. Per `seo-foundations.md` Part C4.

## 9. CTA placement

Phone CTA: exactly 2 times on this page:

1. Sticky mobile header (template handles)
2. Phone block in Section 6.2 (primary CTA, above the fold)

The page is short enough that a final-CTA repeat would be visual padding.

## 10. Page-specific SEO

| Element | Requirement |
|---|---|
| URL | Any unknown URL returns this page |
| HTTP status | 404 (must not be 200) |
| Title tag | `Page Not Found \| [Site Name]` |
| Meta description | None required (404s do not need one) |
| Indexability | Not indexed (HTTP 404 prevents indexing) |
| Phone above fold | Mandatory |

## 11. Anti-patterns (NEVER on the 404 page)

1. **Return HTTP 200 with 404 content (a "soft 404").** This is a Google ranking issue and tells search engines the page exists when it does not.
2. **Hide the phone number below the fold.** Phone is the conversion path. Above the fold is non-negotiable.
3. **Suburb-specific framing.** Generic across all sites. No "Page not found in [Suburb]" — the user does not care which microsite they hit, they want to either call or get to a working page.
4. **Long list of links to other pages.** One link home, one phone CTA. Anything else is decision paralysis on a page where speed to call matters.
5. **Over-apologise.** "Oops!", "Yikes!", "Sorry, we couldn't find that page!" — these waste words. Matter-of-fact wins.
6. **Use humour that does not match the brand.** The brand voice is direct and trustworthy, not chatty. A joke on a 404 reads as off-brand.
7. **Embed a search bar.** The site is small enough (~15-20 pages per microsite) that a search bar is overkill and adds template complexity.
8. **Show error codes, technical details, or stack traces.** "Error 404: Resource not found at endpoint /xyz" is for engineers, not customers.
9. **Cross-link to other sites in the network.** Per `seo-foundations.md` Part C4.
10. **Use em dashes.** Per `brand-voice-guide.md`.
11. **Mention air conditioning, HVAC, electrical, or any non-plumbing service.** Per `content-quality-rules.md` Part C3.
12. **Forget the proper HTTP 404 status header.** This is the template's job, not the Writer's — but it must be verified at build time.

## 12. Per-site rendering

The template renders this page identically on every site, with only the three per-site values from Section 5 injected. The Writer subagent is not invoked.

## Field paths referenced

**From the merged `site-config.json`:**
- `site.siteName`
- `site.trackingPhone`
- `business.hours` (informs the "24/7" copy but the wording is static)
- `transparencyFooter` (rendered in the standard site footer)
